# Contributing to design-discipline

The most valuable contribution is a **new guardrail from a real rejection**.

## Adding a Guardrail

If you use this skill and get feedback that rejects a pattern, add it:

1. Open `skills/design-discipline/reference/guardrails.md`
2. Add a row to the Anti-Patterns table:

| Anti-Pattern | Rejection | Date | Applies To |
|---|---|---|---|
| [What was wrong] | [The actual feedback, raw] | [When] | [Which mediums] |

3. Submit a PR with the quote and context. **Always include the year** in the Date column (e.g., `Feb 26, 2025`).

**Keep the quotes raw.** "It sucks" is more useful than "the visual approach was not well-received." Specificity and honesty are what make the guardrails work.

## Other Contributions

- **Verification improvements** -- new programmatic layout checks (JS snippets), alternative tool support
- **Design knowledge** -- perceptual principles backed by science, not trends
- **Bug fixes** -- if a reference link is broken or a step is unclear

## What We Don't Accept

- **Recipes** -- "use Inter font" or "use 8px grid." The skill is about guardrails (what not to do), not prescriptions (what to do).
- **Trend-based advice** -- "glassmorphism is in." The skill encodes timeless principles.
- **Hype language** -- keep the tone direct. No "revolutionary" or "game-changing."
